# sample-transaction-typeorm

Domain 例

```
WolfDeskは、ヘルプデスクのチケット管理システムをサービスとして提供しています。新興企業が顧客へのサポートを提供する必要がある場合、ウルフデスクのソリューションを使えば、すぐに稼動させることができます。

WolfDeskは、他社とは異なる支払いモデルを採用しています。ユーザーごとに料金を請求するのではなく、テナントが必要な数のユーザーを設定することができ、チャージ期間中に開かれたサポートチケットの数に応じて課金されます。最低料金はなく、毎月のチケット数に応じて自動的にボリュームディスカウントが適用されます。月間500枚以上で10％、750枚以上で20％、1,000枚以上で30％のボリュームディスカウントが適用されます。

ビジネスモデルの悪用を防ぐために、WolfDeskのチケットライフサイクルアルゴリズムは、非アクティブなチケットが自動的にクローズされるようになっており、お客様がさらなるサポートを必要とする際には新しいチケットを開くように促します。さらに、ウルフデスクでは、メッセージを分析して、同じチケットで関係のないトピックが話し合われているケースを検出する不正検出システムを実装しています。

ウルフデスクでは、入居者がサポート関連の作業を効率化できるように、「サポートオートパイロット」機能を実装しています。このオートパイロットは、新規チケットを分析し、テナントのチケット履歴からマッチする解決策を自動的に見つけようとします。この機能により、チケットの寿命をさらに短くすることができ、お客様がさらなる質問のために新しいチケットを開くことを促します。

WolfDeskは、テナントのユーザーを認証するためのすべてのセキュリティ基準と対策が組み込まれており、テナントが既存のユーザー管理システムとの間でシングルサインオン（SSO）を設定することもできます。

管理画面では、チケットのカテゴリーに設定できる値や、サポートしている製品のリストを設定することができます。

WolfDeskでは、新しいチケットをテナントのサポートエージェントの勤務時間中にのみルーティングできるように、各エージェントのシフトスケジュールを入力することができます。

WolfDeskは、最低料金なしでサービスを提供しているため、新しいテナントの導入コストを最小限に抑えるために、インフラを最適化する必要があります。そのために、ウルフデスクはサーバーレスコンピューティングを活用し、アクティブなチケットの操作に応じて計算資源を弾力的に拡張しています。
```
